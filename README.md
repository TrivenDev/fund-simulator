# 模拟基金交易系统

本项目是一个本地运行的基金模拟交易系统。前端负责回测、展示交易记录和资产曲线，Python 后端负责把每次模拟结果保存为 JSON 文件。

## 运行

```powershell
cd fund-simulator
python server.py
```

浏览器打开：

```text
http://127.0.0.1:8765
```

## 功能

- 初始资金固定为 3000 元。
- 默认回测区间为 2022 年 1 月到 2026 年 12 月，也可以在页面自行选择开始月份和结束月份。
- 优先选择 C 类基金。
- 最多同时持有 3 只基金。
- 同一板块最多持有 2 只基金。
- 月度轮动策略：每个月月末卖出上月持仓，再从刚结束月份收益率前 10 的基金中选择标的买入。
- 每次买入后约持有 1 个月，到下一次月末轮动时卖出。
- 页面显示完整交易记录、年度收益和账户资产曲线。
- 点击“更新净值数据”后，后端会通过 AKShare 获取全市场公募基金列表，逐只抓取历史净值，并保存到 `data/funds.json`。
- 前端可以设置本次最多抓取并缓存的基金数量，后端会强制限制不超过 1000 只。
- 点击“保存这次模拟”后，结果保存到 `saved-simulations/`。
- 浏览器 `localStorage` 会保存最近一次模拟，刷新页面后可恢复。
- 点击“开始模拟交易”后，操作日志会显示本次使用的数据来源。

## 数据说明

当前 `data/funds.js` 内置的是可复现的月度净值候选池，作为无法联网或接口不可用时的兜底数据。

点击“更新净值数据”后，系统会优先写入 `data/funds.db` SQLite 本地缓存，并同步导出一份兼容前端旧格式的 `data/funds.json`。之后页面会优先使用 SQLite 缓存；没有可用数据库数据时再回退到 `data/funds.json`，最后回退到 `data/funds.js`。
SQLite 缓存支持断点续抓：每成功抓完一只基金就立即写入数据库；同一区间再次更新时，已有完整缓存的基金会被跳过。

首次全市场更新会比较慢，因为系统需要从 AKShare 的全市场基金目录中逐只抓取历史净值。更新会在独立 worker 进程中串行运行，前端会定时轮询进度。这样可以避免 AKShare 依赖在多线程场景下崩溃时拖垮主 Web 服务。货币、现金管理、理财、同业存单这类不适合用单位净值做 20% 年收益目标回测的基金会自动排除。若结束月份设置到未来，例如 2026-12，但 AKShare 尚无对应月份净值，系统会自动忽略这些未来空月份。

依赖安装：

```powershell
python -m pip install -r requirements.txt
```

字段结构已经固定：

```js
{
  code: "基金代码",
  name: "基金名称",
  type: "A 或 C",
  sector: "板块",
  risk: "风险等级",
  nav: [["YYYY-MM-DD", 1.0000]]
}
```

如果你后续有真实基金净值数据，只要按这个结构生成 `data/funds.json` 即可；没有 `data/funds.json` 时，系统才会回退到 `data/funds.js` 里的 `window.FUND_UNIVERSE`。
